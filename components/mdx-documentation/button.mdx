# Button Component

The `Button` component is a versatile and customizable UI element that provides consistent styling and behavior across your application.

## Import

```tsx

```

</div>


## API Reference

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `variant` | `'default' \| 'destructive' \| 'outline' \| 'secondary' \| 'ghost' \| 'link'` | `'default'` | The visual style of the button |
| `size` | `'default' \| 'sm' \| 'lg' \| 'icon'` | `'default'` | The size of the button |
| `disabled` | `boolean` | `false` | Whether the button is disabled |
| `onClick` | `(event: MouseEvent) => void` | - | Click event handler |
| `children` | `ReactNode` | - | The content of the button |
| `className` | `string` | - | Additional CSS classes |
| `type` | `'button' \| 'submit' \| 'reset'` | `'button'` | The HTML button type |

### TypeScript Interface

```typescript
interface ButtonProps extends React.ButtonHTMLAttributes<HTMLButtonElement> {
  variant?: 'default' | 'destructive' | 'outline' | 'secondary' | 'ghost' | 'link'
  size?: 'default' | 'sm' | 'lg' | 'icon'
  asChild?: boolean
}
```

## Accessibility

The Button component follows WAI-ARIA guidelines:

- ✅ Keyboard accessible (Space and Enter keys)
- ✅ Screen reader friendly
- ✅ Focus management
- ✅ Proper ARIA attributes
- ✅ High contrast support

### Best Practices

> **Tip:** Always provide clear and descriptive button text. Avoid generic labels like "Click here" or "Submit".

> **Note:** When using icon-only buttons, ensure you provide proper `aria-label` attributes for accessibility.

> **Warning:** Don't disable buttons without providing clear feedback about why they're disabled or when they'll become available.
