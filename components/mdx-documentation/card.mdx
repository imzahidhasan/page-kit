# Card Component

A flexible container component for displaying content with consistent styling and layout.

## Installation

```bash
npm install @page-kit/components
```

## Import

```tsx
import {
  Card,
  CardHeader,
  CardTitle,
  CardDescription,
  CardContent,
  CardFooter,
} from "@page-kit/components";
```

## Basic Usage

```tsx
function MyCard() {
  return (
    <Card>
      <CardHeader>
        <CardTitle>Card Title</CardTitle>
      </CardHeader>
      <CardContent>
        <p>Card content goes here.</p>
      </CardContent>
    </Card>
  );
}
```

## API Reference

### Card Props

| Prop        | Type        | Default     | Description            |
| ----------- | ----------- | ----------- | ---------------------- |
| `className` | `string`    | `undefined` | Additional CSS classes |
| `children`  | `ReactNode` | `undefined` | Card content           |

### Card.Header Props

| Prop        | Type        | Default     | Description            |
| ----------- | ----------- | ----------- | ---------------------- |
| `className` | `string`    | `undefined` | Additional CSS classes |
| `children`  | `ReactNode` | `undefined` | Header content         |

### Card.Title Props

| Prop        | Type        | Default     | Description            |
| ----------- | ----------- | ----------- | ---------------------- |
| `className` | `string`    | `undefined` | Additional CSS classes |
| `children`  | `ReactNode` | `undefined` | Title text             |

### Card.Content Props

| Prop        | Type        | Default     | Description            |
| ----------- | ----------- | ----------- | ---------------------- |
| `className` | `string`    | `undefined` | Additional CSS classes |
| `children`  | `ReactNode` | `undefined` | Main content           |

### Card.Footer Props

| Prop        | Type        | Default     | Description            |
| ----------- | ----------- | ----------- | ---------------------- |
| `className` | `string`    | `undefined` | Additional CSS classes |
| `children`  | `ReactNode` | `undefined` | Footer content         |

## Best Practices

- Use cards to group related content together
- Keep card content concise and scannable
- Use consistent spacing and typography
- Consider hover states for interactive cards
- Ensure proper contrast for accessibility
