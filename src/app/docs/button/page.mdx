  code={`

const App = () => (
return (
    <Button>Click Me</Button>
)};


`} />

### Primary

The primary button is used for primary actions and main call-to-actions.

<CodeBlock
  code={'<Button variant="primary">Primary Button</Button>'}
  lang="tsx"
/>

### Secondary

The secondary button is used for secondary actions, providing an alternative to the primary button.

<CodeBlock
  code={'<Button variant="secondary">Secondary Button</Button>'}
  lang="tsx"
/>

### Outline

The outline button has a border but no background, useful for less prominent actions.

<CodeBlock
  code={'<Button variant="outline">Outline Button</Button>'}
  lang="tsx"
/>

### Ghost

The ghost button has no background or border, only showing text and hover effects.

<CodeBlock code={'<Button variant="ghost">Ghost Button</Button>'} lang="tsx" />

### Loading State

Display a loading spinner and disable the button:

<CodeBlock code={"<Button isLoading>Loading Button</Button>"} lang="tsx" />

You can also create custom loading states using Lucide icons:

<CodeBlock
  code={`import { Loader } from "lucide-react";

<Button leftIcon={<Loader className="h-4 w-4 animate-spin" />}>
  Custom Loading
</Button>
;`} lang="tsx" />

### Disabled State

Disable user interaction with the button:

<CodeBlock code={"<Button disabled>Disabled Button</Button>"} lang="tsx" />

## Accessibility

- The button component is built using the native `<button>` element
- It inherits all accessibility features of the native button
- When in loading state, it's automatically disabled to prevent multiple submissions
- It supports keyboard navigation and screen readers
